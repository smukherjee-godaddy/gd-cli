DEVX-70: Surface `app-registry-api` errors with accurate classification and actionable fix hints across every app-lifecycle command (create, update, archive, deploy upload). Server-classified errors — `VALIDATION_ERROR` (now forwarding per-field details), `AUTH_ERROR` / `UNAUTHORIZED`, `FORBIDDEN` / `INSUFFICIENT_PERMISSIONS`, `NOT_FOUND` (including `*_NOT_FOUND` variants), `*_CONFLICT` (duplicate / state / version), and `RATE_LIMIT_EXCEEDED` — are now mapped to dedicated CLI tagged errors (`ForbiddenError`, `NotFoundError`, `ConflictError`, `RateLimitError`, plus an extended `ValidationError.fields`) so the JSON envelope reports the correct top-level `code` and a specific `fix` instead of collapsing everything into `NETWORK_ERROR`. Unknown / future server codes and transport failures still map to `NetworkError`, but now preserve the full HTTP context (status + response body) so agents can still discriminate via `error.details.response.errors[].extensions.code`. The classification logic lives in a shared `src/services/graphql-error.ts` module used by both the application service and the extension presigned-URL service, keeping error semantics consistent across every GraphQL call.
